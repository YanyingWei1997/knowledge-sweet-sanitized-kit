# Tag 白名单

这是 `knowledge-sweet` 的固定 tag 白名单。

更细的 `子类` 选择，请直接看：

- `SUBCATEGORY_WHITELIST.md`

## 使用规则

- 新内容必须先匹配这里的 tag
- 不要为同一种内容临时创建多个近义 tag
- 如果确实需要新增 tag，先记录理由，再补进白名单

## 当前标准 tag

- `教学相关`
  - 适用：课程设计、教案、课件、课堂活动、学生反馈、教学反思
- `科研相关`
  - 适用：论文、文献、研究设计、评审意见、实验思路、学术写作
- `科技学习`
  - 适用：OpenClaw、AI 工具、编程技术、自动化流程、软件配置
- `个人提升`
  - 适用：学习方法、成长复盘、阅读积累、能力训练、个人数据
- `提示词灵感`
  - 适用：Prompt、提示词模板、对话策略、提示词优化想法
- `生活相关`
  - 适用：个人安排、待办、日常记录、非教学非科研的生活信息
- `待整理`
  - 适用：临时速记、临时素材、待归档数据、待分类 Prompt、暂时无法判断的内容

## 不建议单独新建的近义 tag

- `教学` → 统一到 `教学相关`
- `科研` → 统一到 `科研相关`
- `AI学习` / `技术学习` / `工具学习` → 统一到 `科技学习`
- `成长` / `复盘` / `自我提升` → 统一到 `个人提升`
- `Prompt` / `提示词` → 统一到 `提示词灵感`
- `日常` / `生活` → 统一到 `生活相关`

## 特别归类规则

- 教学场景里的 Prompt，不按“Prompt”这个形式归类，而按**用途**归类
- 例如：
  - 教学课件 PPT 生成提示词 → `教学相关 / 课件制作`
  - 科研评审 Prompt → `科研相关 / 论文评审`
  - 通用写作 Prompt 模板 → `提示词灵感 / 通用Prompt`

## 判断顺序

遇到不好归类的内容，按这个顺序判断：

1. 先看是否和教学直接相关
2. 再看是否和科研直接相关
3. 再看是否是技术工具学习
4. 再看是否是个人成长与能力提升
5. 再看是否是 Prompt 灵感
6. 再看是否属于生活相关
7. 实在无法确定时，先归到 `待整理`

## 同项目迭代规则

同一项目下的同一内容，如果发生迭代：

- 以最新修改年月作为正式保存位置
- 历史版本保留在旧年月目录
- 必要时在项目根目录补一个 `README.md` 或 `LATEST.md`，指向当前最新版
