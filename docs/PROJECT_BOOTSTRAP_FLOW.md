# 新建项目固定流程

这个流程用于保证：**每次新建项目时，自动配齐 `LATEST.md`、项目目录和年月目录规则。**

## 标准目标

新项目建立后，至少要有这三层：

`知识库/tag/子类/项目名/`

并且项目根目录下至少有：

- `LATEST.md`

首次内容写入时，再建立：

`知识库/tag/子类/项目名/YYYY/MM/`

## 固定步骤

### Step 1：先选 tag

必须先从 `TAG_WHITELIST.md` 里选一个 tag：

- `教学相关`
- `科研相关`
- `科技学习`
- `个人提升`
- `提示词灵感`
- `生活相关`
- `待整理`

如果拿不准，先不要新建目录，先确认 tag。

### Step 2：确定子类

在确定项目名之前，必须先确定 `子类`。

例如：

- `教学相关 / 讲课设计`
- `教学相关 / 课件制作`
- `科研相关 / 论文评审`
- `科技学习 / 自动化工作流`

### Step 3：确定项目名

项目名要求：

- 尽量短
- 一眼能看懂
- 不和已有项目重名
- 后续迭代还能继续沿用

推荐格式：

- `OpenClaw五机器人群协作`
- `毕业论文评审Prompt`
- `课程评价改革教案`

### Step 4：建立项目根目录

项目根目录格式：

`知识库/tag/子类/项目名/`

例如：

- `知识库/科研相关/论文评审/毕业论文评审Prompt/`

### Step 5：立即放入 `LATEST.md`

项目根目录创建后，**先放 `LATEST.md`**，哪怕当天还只有第一版。

这样做的原因：

- 后续有迭代时，不会忘记补最新版入口
- Obsidian 打开项目时，先看到的是项目导航，不是散乱文件
- 项目即使只有一份内容，也先具备“版本入口”

### Step 6：建立当年当月目录

首次写入内容时，建立：

`知识库/tag/子类/项目名/YYYY/MM/`

例如：

- `知识库/科研相关/论文评审/毕业论文评审Prompt/2026/03/`

### Step 7：把正式文件放入当月目录

例如：

- `知识库/科研相关/论文评审/毕业论文评审Prompt/2026/03/Prompt-毕业论文主审评审.md`

### Step 8：回填 `LATEST.md`

至少补齐：

- tag
- 项目
- 当前最新版本日期
- 当前最新文件
- 本次更新内容

## 迭代规则

同项目出现新版本时：

1. 不覆盖旧年月目录
2. 新版本进入新的年月目录
3. 更新项目根目录的 `LATEST.md`
4. 把“当前最新文件”改成新版本路径

## 最小完成标准

一个新项目创建完成，至少应该有：

- `知识库/tag/子类/项目名/LATEST.md`
- `知识库/tag/子类/项目名/YYYY/MM/首个正式文件.md`

## 推荐检查清单

- tag 是否来自白名单
- 子类是否合理
- 项目名是否清楚且稳定
- 是否已经创建 `LATEST.md`
- 正式文件是否进入年月目录
- `LATEST.md` 是否已经指向最新文件
- Obsidian 是否同步了同样结构

## 当前建议

以后凡是“新建项目”，默认按这个流程执行，不再跳过 `LATEST.md`。
