# knowledge-sweet 映射表示例模板

这份模板用于在正式替换配置前，先梳理目标环境中的机器人资料、账号绑定和群路由关系。

## 一、最小映射表

建议至少填写下表：

| 角色职责 | 飞书显示名 | accountId | agentId | appId | 目标群 ID | requireMention | 工作区/目录 | 备注 |
|---|---|---|---|---|---|---|---|---|
| 主控入口 | `<DISPLAY_NAME_MAIN>` | `knowledge_main` | `main` | `<APP_ID>` | `<YOUR_GROUP_ID>` | `false` | `workspace/knowledge-sweet` | 默认入口 |

## 二、推荐补充字段

如果目标环境比较复杂，建议继续补这些字段：

| 字段 | 含义 | 是否必填 |
|---|---|---|
| `角色职责` | 该机器人在系统中的功能定位 | 必填 |
| `飞书显示名` | 飞书群里看到的机器人名字 | 必填 |
| `accountId` | `openclaw.json` 中账号标识 | 必填 |
| `agentId` | OpenClaw 中绑定的 agent | 必填 |
| `appId` | 飞书应用 ID | 建议填写 |
| `目标群 ID` | 当前接入的群 | 必填 |
| `requireMention` | 是否必须 @ 才响应 | 建议填写 |
| `工作区/目录` | 该账号或 agent 实际使用的工作区 | 建议填写 |
| `备注` | 额外说明，例如“已占用”“不可复用” | 建议填写 |

## 三、最小检查清单

填写映射表后，至少检查这 6 件事：

1. 飞书显示名和 `accountId` 是否一一对应
2. `accountId` 和 `agentId` 是否已经绑定
3. 目标群是否已经挂到某个账号下
4. `main` 是否已存在，是否应复用
5. 是否有旧账号或旧绑定会被覆盖
6. `requireMention` 是否符合当前目标行为

## 四、常见错误示例

### 1. 显示名正确，但 `accountId` 错

现象：

- 群里看到的是对的机器人名字
- 但 `openclaw.json` 里实际写到另一个账号上

### 2. `accountId` 正确，但 `agentId` 错

现象：

- 飞书账号能接消息
- 但实际交给了错误 agent 处理

### 3. 群写对了，但写到了错误账号的 `groups`

现象：

- 目标群无法按预期响应
- 或者消息被错误入口接管

## 五、推荐填写顺序

建议按这个顺序填：

1. 先填飞书显示名
2. 再填 `appId`
3. 再确认 `openclaw.json` 中的 `accountId`
4. 再确认 `agentId`
5. 再补群 ID 和 `requireMention`
6. 最后补工作区和备注
